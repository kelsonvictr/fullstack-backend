application.properties

### ⚙️ Configurando o application.properties

É o arquivo que diz para o Spring como conectar ao banco e outras configurações. Fica em `src/main/resources/application.properties`.

src/main/resources/application.properties

```

# Nome da aplicação
spring.application.name=petshop-api

# Conexão com PostgreSQL
spring.datasource.url=jdbc:postgresql://localhost:5432/petshop
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.datasource.driver-class-name=org.postgresql.Driver

# Hibernate (cria/atualiza tabelas automaticamente)
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

🔑
Atenção na senha! Coloque a senha que você usou ao instalar o PostgreSQL — pode ser `postgres`, `123`, `admin`, etc. Se o app não conectar, o erro vai ser de autenticação.

🐳
Está usando Docker? Se subiu o PostgreSQL com docker-compose, use a senha que você definiu no docker-compose.yml. A URL `localhost:5432` funciona normalmente.

Criar o banco "petshop" no DBeaver

Antes de rodar — sem isso, NADA funciona!

⚠️
Erro mais comum aqui: rodar a aplicação sem ter criado o banco antes. O Spring cria as tabelas automaticamente, mas NÃO cria o banco! É como ter um armário lindo mas esquecer de comprar o armário primeiro. 🪑

🔁 Lembra do início do curso?

No tour do PostgreSQL, criamos o banco `bd_sistema_gestao`. Aqui é a mesma coisa, só muda o nome: `petshop`!

1

2

3

4
O banco aparece na lista. Pronto! Agora pode rodar a aplicação Spring que ela vai conectar e criar as tabelas sozinha via `ddl-auto=update`.

🗺️

A Ordem de Construção — siga sempre nessa sequência!

Antes de codar, entenda a ordem em que cada arquivo deve ser criado

Construir uma API tem uma ordem natural — cada camada depende da anterior. Se você tentar fazer fora de ordem, não vai compilar!

Para cada entidade do PetShop (Tutor e Pet), siga esta sequência de 8 passos:

1

🏷️

Enum

Valores fixos (ex: espécies de pet) — primeiro pacote do projeto

↓

2

🗄️

Model (Entidade)

Classe Java que vira tabela no banco — usa @Entity, @Table, @Id

↓

3

📦

DTOs (Request + Response)

Records que o front envia e recebe — com validações @NotBlank etc

↓

4

🔄

Mapper

Converte DTO ↔ Entidade — métodos estáticos toEntity/toResponseDTO

↓

5

🗃️

Repository

Acesso ao banco — interface que herda JpaRepository

↓

6

📋

Service (Interface)

O contrato — diz QUAIS métodos o CRUD vai ter

↓

7

⚙️

ServiceImpl

A implementação — onde a LÓGICA acontece de verdade

↓

8

🎮

Controller

Endpoints REST — POST, GET, PUT, DELETE expostos pra rede

💡
Por que essa ordem importa? Cada camada depende da anterior. Você não pode fazer o Repository sem o Model, nem o ServiceImpl sem a Interface. Seguir esta ordem evita erros de compilação e te força a pensar passo a passo!

🐾
Estratégia neste desafio: Vamos seguir essa sequência de 1 a 8 PRIMEIRO completinho para o Tutor (do passo 2 ao 8), e SÓ DEPOIS repetir tudo para o Pet. Assim você entende um CRUD inteiro de ponta a ponta antes de começar o próximo. Bem mais didático! 🎯

🧑 2

Model — Tutor.java

A entidade da pessoa dona dos pets

💡 Dica para fazer sozinho

O Tutor segue o MESMO PADRÃO do `Fornecedor` do cap 1! Crie a classe com `@NoArgsConstructor`, `@AllArgsConstructor`, `@Data`, `@Table(name="tutor")` e `@Entity`. Os campos: id (Long, com `@Id` e `@GeneratedValue(strategy = IDENTITY)`), nomeCompleto, email, telefone (todos String, com `@Column(nullable = false)`).

🚨

NÃO clique no toggler ainda! Abra `Fornecedor.java` do cap 1.1 e use como base — basicamente é trocar os nomes dos campos!

👀 Só clique aqui DEPOIS de tentar — Ver código completo ▼

model/Tutor.javaJava

```

package br.com.petshop.api.model;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@NoArgsConstructor
@AllArgsConstructor
@Data
@Table(name = "tutor")
@Entity
public class Tutor {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "nome_completo", nullable = false)
    private String nomeCompleto;

    @Column(name = "email", nullable = false)
    private String email;

    @Column(name = "telefone", nullable = false)
    private String telefone;
}
```

🧑 3

DTOs — TutorRequestDTO e TutorResponseDTO

Records com Bean Validation

💡 Dica para fazer sozinho

Crie `dto/tutor/TutorRequestDTO.java` e `dto/tutor/TutorResponseDTO.java` como records. Para o Request, não inclua o id e use `@NotBlank` em `nomeCompleto`, `email` e `telefone`. Para o Response, inclua o id. É identicamente o que você fez em `ProdutoRequestDTO/ResponseDTO` no cap 2.3.

🚨

Lembre-se: em record o nome dos campos é só o tipo + nome, separado por vírgula. `public record TutorRequestDTO(String nomeCompleto, ...)`.

👀 Só clique aqui DEPOIS de tentar — Ver código completo ▼

dto/tutor/TutorRequestDTO.javaJava

```

package br.com.petshop.api.dto.tutor;

import jakarta.validation.constraints.NotBlank;

public record TutorRequestDTO(

        @NotBlank(message = "O nome completo é obrigatório")
        String nomeCompleto,

        @NotBlank(message = "O e-mail é obrigatório")
        String email,

        @NotBlank(message = "O telefone é obrigatório")
```

════════════════════════════════════════ DESAFIO: TUTOR MAPPER ════════════════════════════════════════

🧑 4

Mapper — TutorMapper.java

Convertendo DTO ↔ Entidade

💡 Dica para fazer sozinho

Use o `ProdutoMapper` do cap 2.4 como referência! Dois métodos estáticos: `toEntity(TutorRequestDTO dto)` que retorna um Tutor, e `toResponseDTO(Tutor tutor)` que retorna um TutorResponseDTO. Aqui é até mais simples que o ProdutoMapper, porque o Tutor não tem entidade relacionada (não tem o "Fornecedor" do meio)!

🚨

Lembrete: no record usamos `dto.nome()` (sem "get"), na entidade com Lombok usamos `tutor.getNome()` (com "get"). Não confunda!

👀 Só clique aqui DEPOIS de tentar — Ver código completo ▼

mapper/TutorMapper.javaJava

```

package br.com.petshop.api.mapper;

import br.com.petshop.api.dto.tutor.TutorRequestDTO;
import br.com.petshop.api.dto.tutor.TutorResponseDTO;
import br.com.petshop.api.model.Tutor;

public class TutorMapper {

    // RequestDTO → Entidade
    public static Tutor toEntity(TutorRequestDTO dto) {

        Tutor tutor = new Tutor();

        tutor.setNomeCompleto(dto.nomeCompleto());
        tutor.setEmail(dto.email());
        tutor.setTelefone(dto.telefone());

        return tutor;
    }

    // Entidade → ResponseDTO
    public static TutorResponseDTO toResponseDTO(Tutor tutor) {

        return new TutorResponseDTO(
                tutor.getId(),
                tutor.getNomeCompleto(),
                tutor.getEmail(),
                tutor.getTelefone()
        );
    }
}
```

════════════════════════════════════════ DESAFIO: TUTOR SERVICE IMPL ════════════════════════════════════════

🧑 7

ServiceImpl — TutorServiceImpl.java

A implementação real — onde a lógica acontece

💡 Dica para fazer sozinho

Coloque a classe no mesmo pacote `service` (ao lado da interface — sem subpasta!). Use `@Service`, `@RequiredArgsConstructor` e `private final TutorRepository`. Implemente os 5 métodos. Nos que buscam por id, use `orElseThrow` com lambda para 404. Na listagem, use `stream().map(TutorMapper::toResponseDTO).toList()`. Igualzinho ao `ProdutoServiceImpl` do cap 2.7 — só sem o segundo Repository!

🚨

Atenção: o TutorServiceImpl é mais simples que o ProdutoServiceImpl porque o Tutor não tem entidade relacionada — só 1 Repository (TutorRepository). Use o ProdutoServiceImpl como base, mas ignore a parte do FornecedorRepository!

👀 Só clique aqui DEPOIS de tentar — Ver código completo ▼

service/TutorServiceImpl.javaJava

```

package br.com.petshop.api.service;

import br.com.petshop.api.dto.tutor.TutorRequestDTO;
import br.com.petshop.api.dto.tutor.TutorResponseDTO;
import br.com.petshop.api.mapper.TutorMapper;
import br.com.petshop.api.model.Tutor;
import br.com.petshop.api.repository.TutorRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpStatus;
import org.springframework.stereotype.Service;
import org.springframework.web.server.ResponseStatusException;

import java.util.List;

@Service
@RequiredArgsConstructor
public class TutorServiceImpl implements TutorService {

    private final TutorRepository tutorRepository;

    @Override
    public TutorResponseDTO criarTutor(TutorRequestDTO dto) {

        Tutor tutor = TutorMapper.toEntity(dto);
        tutorRepository.save(tutor);

        return TutorMapper.toResponseDTO(tutor);
    }

    @Override
    public List<TutorResponseDTO> listarTodosTutores() {

        return tutorRepository.findAll()
                .stream()
                .map(TutorMapper::toResponseDTO)
                .toList();
    }

    @Override
    public TutorResponseDTO buscarTutorPorId(Long id) {
```

```
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@NoArgsConstructor
@AllArgsConstructor
@Data
@Table(name = "pet")
@Entity
public class Pet {
```

```
package br.com.petshop.api.mapper;

import br.com.petshop.api.dto.pet.PetRequestDTO;
import br.com.petshop.api.dto.pet.PetResponseDTO;
import br.com.petshop.api.model.Pet;
import br.com.petshop.api.model.Tutor;

public class PetMapper {

    // RequestDTO + Tutor → Pet (entidade)
    public static Pet toEntity(PetRequestDTO dto, Tutor tutor) {

        Pet pet = new Pet();

        pet.setNome(dto.nome());
        pet.setRaca(dto.raca());
        pet.setIdade(dto.idade());
        pet.setEspecie(dto.especie());
        pet.setTutor(tutor);   // objeto completo, não o ID!

        return pet;
    }

    // Pet (entidade) → ResponseDTO
    public static PetResponseDTO toResponseDTO(Pet pet) {

        return new PetResponseDTO(
                pet.getId(),
                pet.getNome(),
                pet.getRaca(),
                pet.getIdade(),
                pet.getEspecie(),
                pet.getTutor().getId(),
                pet.getTutor().getNomeCompleto()
        );
    }
}
```

```
package br.com.petshop.api.service;

import br.com.petshop.api.dto.pet.PetRequestDTO;
import br.com.petshop.api.dto.pet.PetResponseDTO;
import br.com.petshop.api.mapper.PetMapper;
import br.com.petshop.api.model.Pet;
import br.com.petshop.api.model.Tutor;
import br.com.petshop.api.repository.PetRepository;
import br.com.petshop.api.repository.TutorRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.http.HttpStatus;
import org.springframework.stereotype.Service;
import org.springframework.web.server.ResponseStatusException;

import java.util.List;

@Service
@RequiredArgsConstructor
public class PetServiceImpl implements PetService {

    private final PetRepository petRepository;
    private final TutorRepository tutorRepository;

    @Override
    public PetResponseDTO criarPet(PetRequestDTO dto) {

        Tutor tutor = tutorRepository.findById(dto.tutorId())
                .orElseThrow(() -> new ResponseStatusException(
                        HttpStatus.NOT_FOUND, "Tutor não encontrado"));

        Pet pet = PetMapper.toEntity(dto, tutor);
        petRepository.save(pet);

        return PetMapper.toResponseDTO(pet);
    }

    @Override
    public List<PetResponseDTO> listarTodosPets() {

        return petRepository.findAll()
                .stream()
                .map(PetMapper::toResponseDTO)
                .toList();
    }

    @Override
    public PetResponseDTO buscarPetPorId(Long id) {

        Pet pet = petRepository.findById(id)
                .orElseThrow(() -> new ResponseStatusException(
                        HttpStatus.NOT_FOUND, "Pet não encontrado"));

        return PetMapper.toResponseDTO(pet);
    }

    @Override
    public PetResponseDTO atualizarPetPorId(Long id, PetRequestDTO dto) {

        Pet pet = petRepository.findById(id)
                .orElseThrow(() -> new ResponseStatusException(
                        HttpStatus.NOT_FOUND, "Pet não encontrado"));

        Tutor tutor = tutorRepository.findById(dto.tutorId())
                .orElseThrow(() -> new ResponseStatusException(
                        HttpStatus.NOT_FOUND, "Tutor não encontrado"));

        pet.setNome(dto.nome());
        pet.setRaca(dto.raca());
        pet.setIdade(dto.idade());
        pet.setEspecie(dto.especie());
        pet.setTutor(tutor);

        petRepository.save(pet);

        return PetMapper.toResponseDTO(pet);
    }

    @Override
    public void deletarPetPorId(Long id) {

        Pet pet = petRepository.findById(id)
                .orElseThrow(() -> new ResponseStatusException(
                        HttpStatus.NOT_FOUND, "Pet não encontrado"));

        petRepository.deleteById(pet.getId());
    }
}
```

✨ Por que o PetServiceImpl precisa do TutorRepository?

Mesmo motivo do `ProdutoServiceImpl` precisar do `FornecedorRepository`! O DTO traz só o `tutorId` (um Long), mas a entidade Pet precisa do objeto Tutor inteiro pra gravar o relacionamento. Por isso buscamos o Tutor antes — e se ele não existir, devolvemos 404 Not Found automaticamente via `orElseThrow`!
