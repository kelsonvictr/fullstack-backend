🦋 O que é o Insomnia?

O Insomnia é uma ferramenta para testar APIs. Ele permite enviar requisições HTTP (GET, POST, PUT, DELETE) para o seu backend e ver a resposta — sem precisar de um frontend pronto. É como um "controle remoto" da sua API!

Analogia: Controle Remoto da TV

Imagine que sua API é uma TV. O frontend React é o controle remoto bonito que o usuário vai usar. Mas durante o desenvolvimento, você precisa de um controle técnico para testar todos os botões — esse é o Insomnia! Ele te permite apertar cada botão (endpoint) e ver se funciona, antes de construir o controle bonito. 📺🎮

📥 Instalando o Insomnia

  1. Acesse insomnia.rest/download
  2. Baixe e instale normalmente
  3. Abra o Insomnia e crie um novo projeto chamado "Sistema Gestao"

📂 Organizando com Pastas

Uma boa prática é organizar os endpoints em pastas por entidade. Assim fica fácil encontrar o que você precisa:

1. Criar pastas

Clique com botão direito no painel esquerdo → "New Folder". Crie uma pasta para cada entidade: Fornecedor, Produto, Cliente.

2. Criar requests dentro das pastas

Dentro de cada pasta, crie os requests (um para cada endpoint). O Insomnia mostra o método HTTP com cor ao lado do nome — igual na imagem abaixo!

📁 Estrutura no Insomnia:

📁 Produto
📁 Cliente
📁 Fornecedor
PUT atualizarFornecedorPorId
GET buscarFornecedorPorId
GET listarTodosFornecedores
POST criarFornecedor
💡
Dica: Nomeie cada request com o nome do método do Controller. Assim quando você vê "criarFornecedor" no Insomnia, sabe exatamente qual método está testando no código!

🔗 Configurando a Base URL

Para não digitar http://localhost:8080 em toda requisição, crie uma variável de ambiente:

1. Acessar Environment

Clique em "Base Environment" no painel esquerdo (ou no ícone de engrenagem ⚙️)

2. Adicionar variável

Adicione a variável baseUrl com o valor http://localhost:8080

3. Usar nas requisições

Na URL de cada request, use _.baseUrl seguido do endpoint. Exemplo: _.baseUrl/fornecedores

Se precisar mudar a URL base (ex: quando for pra produção), é só alterar em um lugar só — a variável de ambiente. Todos os requests atualizam automaticamente! 🎯

🚀 Montando cada Request — Passo a Passo

POST Criar Fornecedor

URL: _.baseUrl/fornecedores
Method: POST
Body: Clique na aba "Body" → selecione "JSON"
JSON 
{
  "nome": "Distribuidora ABC",
  "cnpj": "12.345.678/0001-99",
  "email": "contato@abc.com",
  "telefone": "(11) 99999-0001",
  "cep": "01310-100",
  "logradouro": "Avenida Paulista",
  "numero": "1000",
  "bairro": "Bela Vista",
  "cidade": "São Paulo",
  "uf": "SP"
}
✅
Resposta esperada: 201 Created. Não retorna body — mas o fornecedor foi salvo no banco!

GET Listar Todos Fornecedores

URL: _.baseUrl/fornecedores
Method: GET
Body: Nenhum — GET não envia body!
✅
Resposta esperada: 200 OK com um array JSON contendo todos os fornecedores cadastrados.

GET Buscar Fornecedor Por ID

URL: _.baseUrl/fornecedores/1 (troque o 1 pelo ID desejado)
Method: GET
✅
Resposta esperada: 200 OK com o JSON de um fornecedor específico.

PUT Atualizar Fornecedor

URL: _.baseUrl/fornecedores/1
Method: PUT
Body: JSON com os dados atualizados (mesmo formato do POST)
✅
Resposta esperada: 204 No Content. Sem body na resposta, mas o fornecedor foi atualizado!

DELETE Deletar Fornecedor

URL: _.baseUrl/fornecedores/1
Method: DELETE
Body: Nenhum
✅
Resposta esperada: 204 No Content. Fornecedor removido do banco!
⚠️
Ordem de teste recomendada: Primeiro faça um POST para ter dados no banco, depois GET para listar, depois PUT para editar, e por último DELETE para remover. Se tentar GET sem ter criado nada, vai voltar lista vazia!
